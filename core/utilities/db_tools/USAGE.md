# db_tools — Usage Guide

## Setup

Set DB credentials as environment variables (defaults match the project DB):

```bash
export DB_HOST="34.70.79.163"
export DB_PORT="3306"
export DB_USER="sa"
export DB_PASSWORD="your-password"
export DB_NAME="agentichub"
```

---

## Log an agent operation

Call `log_agent_op` after a successful agent run to insert a row into `AGENT_OP`:

```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parents[N]))  # reach repo root

from core.utilities.db_tools.agent_op import log_agent_op

log_agent_op(
    identifier="SCRUM-5",               # ticket ID or document stem
    agent="Requirements",               # agent name
    artifact="GITHUB",                  # output file destination
    request_type="new development",     # request type
    filename="req_SCRUM-5_20260413_10.md",
    path="/home/.../output",
)
```

If the DB is unreachable the call is a no-op — it logs a warning and never raises.

## AGENT_OP table columns

| Column       | Source                                      |
|--------------|---------------------------------------------|
| IDENTIFIER   | ticket_id or document stem                  |
| AGENT        | agent name                                  |
| ARTIFACT     | output file destination                     |
| REQUEST_TYPE | request_type (title-cased)                  |
| FILENAME     | output filename                             |
| FILE_TYPE    | derived from filename extension (e.g. "MD") |
| PATH         | output directory path                       |
| CREATED_TS   | auto-generated by DB default                |

## Phase 5: Testing

1. **Connectivity Test:** Verify Cloud Function can connect to MySQL source and retrieve sample data from EMPLOYEES table - Expected: Successful connection and data retrieval
2. **Schema Validation Test:** Load test data and verify all 14 columns map correctly with proper data types - Expected: All columns present with correct INT64, STRING, DATE, NUMERIC, TIMESTAMP types
3. **Primary Key Test:** Insert duplicate employee_id records - Expected: BigQuery handles duplicates according to configured strategy
4. **Partitioning Test:** Insert records with different hire_date values spanning multiple months - Expected: Data correctly distributed across monthly partitions
5. **Self-Referencing Relationship Test:** Verify manager_id references valid employee_id values - Expected: Referential integrity maintained for manager-employee relationships
6. **Null Handling Test:** Insert records with null values in nullable columns (phone_number, commission_pct, manager_id, department_id, updated_date) - Expected: Null values accepted in nullable columns, rejected in non-nullable columns
7. **End-to-End Pipeline Test:** Execute complete DAG from Cloud Composer - Expected: Successful data extraction, transformation, and loading with monitoring alerts
8. **Error Recovery Test:** Simulate network failure during data extraction - Expected: Retry logic activates with exponential backoff, eventual success or proper error notification

- **Data Consistency**: Use timestamp-based incremental extraction with overlap buffer

## Phase 1: Source Connectivity and Extraction Setup

1. Verify connectivity to MySQL database at 34.70.79.163:3306 using username 'sa' and test access to EMPLOYEES table in agentichub database
2. Create Cloud Function for MySQL data extraction using pymysql connector
3. Configure Secret Manager entries for MySQL credentials
4. Implement timestamp-based incremental extraction logic for EMPLOYEES table
5. Add retry logic and error handling for database connection failures
6. Create GCS bucket for staging extracted data files

## Phase 2: Data Processing and Transformation

1. Implement lightweight data transformation in Cloud Function to map EMPLOYEES columns to target schema
2. Apply data type conversions (INT64, STRING, DATE, NUMERIC, TIMESTAMP)
3. Handle nullable fields appropriately (phone_number, commission_pct, manager_id, department_id, updated_date)
4. Add record-level timestamps (created_date, updated_date)
5. Stage transformed data to GCS in CSV/JSON format
6. Implement data validation checks for primary key uniqueness and required fields

## Phase 3: BigQuery Integration

1. Create BigQuery dataset for staging layer
2. Create stg_employees table with defined schema including partitioning by hire_date (monthly) and clustering by employee_id, department_id
3. Configure BigQuery load jobs to ingest data from GCS staged files
4. Implement self-referencing relationship validation for manager_id to employee_id
5. Set up load job monitoring and error handling
6. Configure automated cleanup of processed GCS files

## Phase 4: Orchestration and Monitoring

1. Create Cloud Composer DAG to orchestrate the complete pipeline workflow
2. Configure pipeline scheduling and dependency management
3. Implement Cloud Monitoring alerts for pipeline failures and data quality issues
4. Set up logging and observability for each pipeline component
5. Create dashboards for pipeline performance metrics
6. Configure alerting for connection failures and data consistency issues

## Phase 5: Testing

1. **Connectivity Test**: Verify successful connection to MySQL database at 34.70.79.163:3306 with expected return of EMPLOYEES table schema
2. **Data Extraction Test**: Extract sample EMPLOYEES data and verify all 14 target columns are correctly mapped and transformed
3. **Primary Key Test**: Load data with duplicate employee_id values and verify primary key constraint enforcement fails appropriately
4. **Nullable Fields Test**: Load records with null values in phone_number, commission_pct, manager_id, department_id, updated_date and verify they are accepted
5. **Partitioning Test**: Load EMPLOYEES data spanning multiple months and verify records are correctly partitioned by hire_date
6. **Self-Reference Test**: Verify manager_id values correctly reference existing employee_id values in the same table
7. **Incremental Load Test**: Perform initial load, then incremental load with timestamp overlap buffer and verify no duplicate records
8. **Error Handling Test**: Simulate network failure during extraction and verify retry logic executes correctly
9. **End-to-End Test**: Execute complete pipeline from MySQL extraction through BigQuery load and verify data integrity and completeness

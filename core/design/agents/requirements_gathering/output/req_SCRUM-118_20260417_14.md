## Objective
Order data needs to be available in two ways — real-time order status events streamed from Google Pub/Sub for live dashboards, and a daily batch reconciliation from the PostgreSQL operational database to correct any missed or late-arriving events. Both paths write to the same BigQuery dataset using a Lambda architecture.

## Business Context
The Order Analytics Platform enables real-time visibility into order lifecycle status for live operational dashboards while guaranteeing data completeness and accuracy through daily batch reconciliation. The Lambda architecture supports both sub-minute event-driven analytics and reliable full historical order records for business reporting, with stakeholder John as the primary owner.

## Functional Requirements
- Consume order status events (order_placed, order_shipped, order_delivered, order_cancelled) from Pub/Sub topic orders.events via Dataflow and write to the order_events table in real-time
- Perform daily incremental load from PostgreSQL orders and order_items tables using the updated_at watermark
- Both streaming and batch paths write to the verizon_orders_deah dataset in BigQuery project verizon-data
- Batch path reconciles any missed or late-arriving events not captured by the streaming path
- Deduplicate records on event_id and order_id primary keys across both streaming and batch paths
- Partition all BigQuery target tables by event_date and order_date
- Log pipeline execution results to pipeline_audit.pipeline_run_log for every run

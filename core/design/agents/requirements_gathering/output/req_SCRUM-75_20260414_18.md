## Objective
Address performance degradation from heavy analytical queries on transactional database and eliminate inconsistent data across teams by creating centralized analytics store

## Business Context
Heavy analytical queries on transactional database are causing performance degradation and production incidents. Teams are experiencing inconsistent data due to lack of centralized analytics store. Need to create single source of truth for analytics to eliminate query timeouts.

- Automated reconciliation query run post-load with results logged to pipeline_audit.pipeline_run_log

## Acceptance Criteria
- Row count in BigQuery raw table equals row count in source table at time of extraction
- Automated reconciliation query run post-load with results logged to pipeline_audit.pipeline_run_log

## Objective
Build a full historical load followed by daily incremental pipeline to extract products, categories, suppliers, and inventory tables from a PostgreSQL operational database to BigQuery for downstream product performance reporting and inventory trend analysis.

## Business Context
Product catalog and inventory data currently resides in a PostgreSQL operational database and is entirely unavailable for analytics. Surfacing this data in BigQuery will unblock product performance reporting and inventory trend analysis for stakeholder John, enabling data-driven decisions on catalog and stock management.

## Functional Requirements
- Perform a full historical load of all 4 source tables (products, categories, suppliers, inventory) on the first pipeline run
- Perform daily incremental sync using updated_at watermark for products and inventory tables
- Perform a full daily reload for categories and suppliers as small reference tables
- Apply UPSERT logic on primary key in BigQuery to handle late-arriving updates
- Airflow DAG must complete before 03:00 AM UTC every day
- Load data into BigQuery project verizon-data, dataset verizon_catalog_deah, into tables mirroring source names
- Log all pipeline run results to pipeline_audit.pipeline_run_log
- Perform COUNT(*) reconciliation between source and target after every run

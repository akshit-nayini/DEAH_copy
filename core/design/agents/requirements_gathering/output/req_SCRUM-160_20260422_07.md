## Objective
Design and implement a reliable and scalable pipeline to load MySQL tables from the agentichub database into Google BigQuery (verizon_data_dea dataset), supporting a one-time full historical load for all identified tables followed by ongoing scheduled incremental loads (hourly or daily per table) to underpin analytics, reporting, and long-term data storage for the 5gcore programme.

## Business Context
The 5gcore programme requires consolidated analytics, reporting, and long-term data storage backed by MySQL source data in the agentichub database. Stakeholders (led by Yash) require independent visibility into data quality, load success, and audit logs without engineering involvement. The pipeline covers three workstreams — historical load, scheduled incremental load, and end-to-end monitoring with audit logging — and is currently in draft status pending stakeholder review.

## Functional Requirements
- Perform a one-time full historical load of all identified MySQL (agentichub) tables into BigQuery dataset verizon_data_dea with 100% data completeness before any incremental loads are activated
- Implement scheduled incremental loads running hourly or daily per table using timestamp or primary key-based change tracking to detect new and updated records
- Execute all load logic via stored procedure sp_mysqltobq_load.sql with merge key on employee_id and updated_date
- Block the incremental load pipeline from starting until full historical load validation is confirmed and passes
- Enforce a 60-day data retention policy on the target BigQuery table
- Perform schema validation before each load with mismatch logging and a documented strategy for handling new MySQL columns
- Perform row count reconciliation between MySQL and BigQuery after every load
- Confirm primary key uniqueness post-load and verify null and datatype constraints against schema definitions
- Maintain a complete audit log for every load execution capturing status, record counts, and timestamps
- Provide stakeholder-accessible dashboards or log views enabling independent review of load success and execution history without engineering involvement
- Send alerts promptly on any pipeline failure or data quality violation
- Support an initial data volume of 5 GB with ongoing growth of 1 GB/month

## Objective
Build a real-time streaming pipeline using Dataflow (Apache Beam) to consume user clickstream events (page views, clicks, session activity) from a Confluent Kafka cluster and write them to BigQuery for live user behaviour dashboards and funnel analysis, with events available within 60 seconds of source publish and no data loss.

## Business Context
User clickstream events are currently published to a Confluent Kafka cluster but are unavailable for analytics. Delivering these events into BigQuery within 60 seconds will power real-time Looker dashboards and funnel analysis, enabling live visibility into user behaviour across page views, clicks, and session activity.

## Functional Requirements
- Consume events in real-time from 3 Confluent Kafka topics (clickstream.page_views, clickstream.user_events, clickstream.sessions) using Dataflow Streaming
- Parse JSON event payloads and map fields to the corresponding BigQuery table schema
- Deduplicate events on event_id within a 10-minute sliding window
- Partition BigQuery target tables by event_timestamp (DATE)
- Route unparseable or schema-invalid events to a dead-letter Kafka topic without interrupting the pipeline
- Log pipeline run results to pipeline_audit.pipeline_run_log

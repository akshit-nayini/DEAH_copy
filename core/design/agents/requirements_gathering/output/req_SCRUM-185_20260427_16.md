## Objective
Design and implement a reliable and scalable MySQL to BigQuery pipeline to support analytics, reporting, and long-term data storage, including a one-time full historical load for all identified tables and ongoing scheduled incremental loads (hourly or daily per table) thereafter.

## Business Context
The analytics team requires reliable access to MySQL source data in Google BigQuery to support reporting and long-term data storage. All agreed-upon tables must be present in BigQuery with complete historical data and reliable scheduled data refreshes so the analytics team has full confidence in the data. Stakeholder-sponsored initiative owned by Shruthi B, currently in draft pending stakeholder review.

## Functional Requirements
- Perform a one-time full historical load from MySQL to BigQuery for all identified tables without data loss
- Build and deploy a scheduled incremental data load pipeline from MySQL to BigQuery, running hourly or daily depending on per-table configuration
- Use timestamp-based or primary key-based change tracking to identify and load only new or changed records each incremental cycle
- Gate incremental loads so they cannot begin until the full historical load has been validated and confirmed complete
- Implement pre-load schema validation that detects type mismatches, null handling inconsistencies, and field mapping issues between MySQL and BigQuery schemas
- Define and implement a strategy for handling new columns that appear in the MySQL source (e.g., alert and hold, or auto-map with review)
- Implement per-load data quality checks covering row count reconciliation, primary key uniqueness, and data type constraint verification
- Maintain audit logs for all load executions capturing execution time, record counts, validation results, and any errors
- Provide stakeholder visibility into load success through accessible logs or a dashboard without requiring engineering involvement
- Apply merge key logic using employee_id and updated_date for upsert/merge operations via stored procedure sp_mysqltobq_load.sql
